1. Architecture

✅ Good
⚪
✅ Lazy Loading — 10 components via Suspense
lazy-widgets.ts يحمّل 10 مكونات تقيلة (PortfolioTracker, TradingJournal, OsoulArchitect, إلخ) عند الحاجة فقط. كل widget داخل <Suspense fallback={<WidgetSkeleton />}>. ممتاز للـ bundle size.
📁 src/components/lazy-widgets.ts
🔵
❌ DashboardLayout.tsx — Orphaned Component
ملف DashboardLayout.tsx معرف وقاعد يستخدم grid layout بس ما ينستخدم في أي مكان. مكانه في السورس كود بدون فائدة.
🔧 التوصية: احذف الملف لأنه dead code. أو استخدمه إذا كان مخطط له مستقبلًا.
📁 src/components/DashboardLayout.tsx:1-13
⚪
✅ Supabase Hardcoded Keys
Anon key موجود hardcoded في supabase.ts. للـ static export على GitHub Pages هذي مقبولة لأن anon key آمن مع Row Level Security، بس الأفضل تحطها في .env.local عشان الـ hygiene.
🔧 التوصية: انقل الـ keys إلى NEXT_PUBLIC_SUPABASE_URL و NEXT_PUBLIC_SUPABASE_ANON_KEY في ملف env.
📁 src/lib/supabase.ts:3-4
⚪
✅ Tree-shaking for lucide-react
next.config.js يضمّن modularizeImports للـ lucide-react. هذا يضمن أن الـ bundle ما يحمل icons غير المستخدمة.
📁 next.config.js:10-14

2. UI/UX

⚠️ Needs Work
🔴
❌ Light Mode Broken — Hardcoded Dark Colors
أكبر مشكلة في الـ dashboard حاليًا. الـ data-theme="light" يحط CSS variables للـ light mode بس كل المكونات تستخدم ألوان hardcoded خاصة بالـ dark mode: bg-zinc-900/30, bg-zinc-900/90, bg-white/[0.06], text-white, border-white/[0.06].

النتيجة: إذا شخص غير محمد فتح الـ dashboard في light mode، بعض الأقسام تختفي أو تصبح غير مقروءة لأن النص أبيض على خلفية فاتحة.

المكان: هذه الأنماط موجودة في كل ملف: CommandCenter.tsx, SummaryCard.tsx, AuthGuard.tsx, Onboarding.tsx, كل desks, كل widgets.
🔧 التوصية: استخدم Tailwind 4 dark mode classes. بدال bg-zinc-900/30 استخدم class="dark:bg-zinc-900/30 bg-white/70". أو استخدم CSS variables custom لكل الخلفيات. هذا تصليح كبير—يحتاج مسح كل الملفات الـ 47.
البديل السريع: حول data-theme="light" إلى data-theme="dark" افتراضيًا وشيل خيار التبديل إلى light mode مؤقتًا لحين التصليح.
📁 جميع الملفات (47 ملف) — ⚠️ هذه مشكلة منتشرة
🟡
❌ RTL Support — Non-Functional Despite Arabic Content
الـ layout.tsx يستخدم dir="auto" و lang="en". في المقابل، الـ onboarding فيه محتوى عربي كامل، و TodayWidget فيه أيام عربية (الأحد، الإثنين...) وتحيات عربية (صباح الخير، مساء الخير).

dir="auto" ما يشتغل بشكل صحيح مع التطبيقات أحادية الاتجاه. النص العربي يظهر بشكل سيئ في الوضع الحالي لأن المحاذاة والـ padding ما صممت للـ RTL.
🔧 التوصية: (1) غير dir="auto" إلى dir="rtl" في layout. (2) أضف lang="ar". (3) اختبر كل المكونات وشوف إذا في breakages في الـ layout. (4) استخدم start/end بدال left/right في Tailwind.
📁 src/app/layout.tsx:38, src/components/Onboarding.tsx, src/components/TodayWidget.tsx
🟡
✅ FinancialDesk — Dead Buttons
زرارين "Tracker" و "Osoul" في FinancialDesk عندهم onClick={() => {}} — ما يسوون شي. يخرب تجربة المستخدم لأن الزرار يظهر نشط بس ما يشتغل.
🔧 التوصية: شيل الأزرار أو حط navigation حقيقي يبدّل بين views داخل الـ Financial desk.
📁 src/components/desks/FinancialDesk.tsx:22-34
🔵
✅ Stagger Animation — Limited to 12 Items
.animate-stagger يعمل فقط لأول 12 عنصر (مكتوب يدويًا nth-child(1) إلى nth-child(12)). إذا زادت العناصر، الـ animation يوقف.
🔧 التوصية: استخدم @for loop أو JS inline بدال nth-child اليدوي.
📁 src/app/globals.css:405-416
🔵
✅ Auth Screen — "Admin Override" Confusing Label
كلمة "Admin Override" تحسّس المستخدم العادي أنه مو مرحب به. الأفضل "Sign in with Email" عشان واضح.
🔧 التوصية: غير label إلى "Sign in with Email" أو "بريد إلكتروني" للعربية.
📁 src/components/AuthGuard.tsx:238-248

3. Functionality

✅ Good
🔵
❌ No Error Handling on Supabase Queries
كثير من مكونات الـ widgets تتعامل مع Supabase بدون معالجة أخطاء .catch(). مثلاً OverviewDesk.tsx يسوي .then(({ data }) => {...}) بدون أي error handling إذا فشل الاتصال. الـ Guest feature يحمي من هذا جزئيًا بس لو Supabase عطى error، المكون يقعد في loading state للأبد.
🔧 التوصية: أضف .catch() لكل استعلام Supabase مع fallback data أو error message للمستخدم.
📁 src/components/desks/OverviewDesk.tsx:29-65 (وجميع المكونات الأخرى)
🟡
❌ FAB All Actions Navigate to Operating Desk — Wrong UX
كل زرار في الـ FAB (Add Task, Log Shift, Mark Habit, Quick Note) يودّي على Operating desk. المستخدم يتوقع أن كل زر يودي على المكان المناسب أو يفتح modal مباشر. الوضع الحالي يخلي الـ FAB غير مجدي—كل الأزرار تسوي نفس الشي.
🔧 التوصية: بدال التنقل، افتح modal مباشر لكل إجراء:
• Add Task → فتح modal إضافة مهمة
• Log Shift → فتح modal تسجيل مناوبة
• Mark Habit → فتح modal تعليم عادة
• Quick Note → فتح modal ملاحظة سريعة
📁 src/components/CommandCenter.tsx:276-281, src/components/QuickActionFab.tsx:53-82
🟡
❌ CashRunway Uses Hardcoded 80% Formula
CashRunway.tsx يحسب income * 0.8 بدال استخدام savings_goal_percent الفعلي من قاعدة البيانات. هذا يعرض نسبة استخدام غير دقيقة للميزانية.
🔧 التوصية: استخدم data.savings_goal_percent بدال 0.8 hardcoded.
📁 src/components/CashRunway.tsx:41
🔵
⚠️ Habit Duplicated Definitions
نفس الـ 10 habits معرفة في utils.ts (HABITS array) و HabitTracker.tsx. إذا تغيرت قائمة العادات في مكان، الثاني ما يتغير—يسبب inconsistency.
🔧 التوصية: استخدم الـ HABITS من utils.ts فقط (HabitTracker حالياً يعرف عادات خاصة فيه).
📁 src/lib/utils.ts:60-71, src/components/HabitTracker.tsx:16-27

5. Ease of Use

⚠️ Needs Work
🟡
❌ No Keyboard Shortcut Reference
محمد مو programmer — ما يعرف أن 1-4 يبدّل بين المكاتب أو ? يفتح الـ command palette. الـ ⌘K مذكور بشكل خفيف في الـ header. لا يوجد onboarding للـ keyboard shortcuts.
🔧 التوصية: أضف زر "?" دائري في الزاوية أو اختصار في الـ header بعنوان "Keyboard Shortcuts" يظهر modal فيه كل الاختصارات:
1-4 → تغيير المكتب
⌘K / ? → فتح الـ palette
Esc → إغلاق كل شي
📁 src/components/CommandCenter.tsx:218-259
🔵
⚠️ Onboarding — No Replay Option
الـ onboarding يظهر مرة واحدة فقط (localStorage). إذا المستخدم نسي شي أو شخص آخر استخدم الـ dashboard، ما في طريقة يعيد الـ tour.
🔧 التوصية: أضف زر "Show Onboarding" في الـ header أو الـ settings. أضف localStorage.removeItem('lcc-onboarded') من command palette.
📁 src/components/Onboarding.tsx
🟡
❌ No Feedback After Guest Mode Authentication
المستخدم في Guest mode يضغط "Authenticate" — ما يصير شي واضح. الـ guest mode banner يختفي بس ما في تأكيد للمستخدم أن authentication تم بنجاح.
🔧 التوصية: أضف toast/notification بعد successful auth. أو غير الـ localStorage.removeItem إلى تأكيد مع رسالة.
📁 src/components/AuthGuard.tsx:327-335
🔵
✅ Touch targets — Good Minimum Size
كل الأزرار تستخدم min-h-[44px] و min-h-[48px] ممتازة لـ accessibility على الموبايل.

6. Practicality

✅ Good
🟡
❌ Data Freshness — No Pull-to-Refresh
الـ dashboard يعرض بيانات Supabase بدون طريقة سهلة للمستخدم لتحديثها. محمد يحتاج يروح صفحة ثانية ويرجع عشان يشوف التحديثات. بعض الـ widgets تسوي fetch مرة واحدة فقط (useEffect بدون interval).
🔧 التوصية: (1) أضف زر Refresh في header الـ dashboard. (2) استخدم useCallback مع refetch() pattern لكل widget. (3) أضف auto-refresh كل 5 دقائق (NotificationBell يسويها، الباقي لا).
📁 جميع الـ widgets
🔵
⚠️ Mixed Data Sources — localStorage + Supabase
بعض الإعدادات في localStorage (theme, guest mode, onboarding) وبعضها في Supabase. هذا يخلق confusion: إذا المستخدم مسح cache، يفقد بعض الـ preferences.
🔧 التوصية: انقل الـ preferences إلى Supabase user profile table وتحمّل عند login. الـ localStorage يكون cache فقط.
⚪
✅ Core Coverage — ممتاز
الـ dashboard يغطي: المناوبات، العادات، المهام، المالية (محفظة + أسهم + تداول)، المشاريع، القضايا القانونية، الفواتير، الميزانية، الضمانات. هذا تكامل ممتاز لاحتياجات محمد اليومية.
